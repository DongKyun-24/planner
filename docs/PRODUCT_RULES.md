# Product Rules

이 문서는 웹이 사용자에게 어떻게 보여야 하는지 정리합니다.

## 공통 개념

### Count와 마감일 D-day는 다르다

- `마감일 D-day`는 특정 날짜까지 남은 일수를 본다.
- `Count`는 시작일 기준으로 반복되는 기준일을 본다.
- Count는 시작일을 `1일`로 계산한다.

### Count 기준

- `중요 D-day 주기`에 쉼표로 여러 숫자를 넣는다.
- 예: `100, 365`
- 의미:
  - `100일, 200일, 300일 ...`
  - `365일, 730일, 1095일 ...`
- Count는 다음 기준일까지 `10일 이내`일 때 Today/체크 쪽에 보인다.

## 체크 패널 규칙

- 체크 패널은 `Task / 마감일 / Count` 3분할로 간다.
- Count는 더 이상 마감일 목록 안에 섞여 보이지 않는다.
- `추가`는 현재 탭 기준으로 동작한다.
  - Task면 Task 추가
  - 마감일이면 마감일 추가
  - Count면 Count 추가

## Count 규칙

- Count는 일반 일정 본문과 분리된 개념으로 보여야 한다.
- 브라우저 `prompt`가 아니라 전용 모달에서 생성한다.
- Today/체크에서 마감일 D-day와 Count가 구별되게 보여야 한다.
- Count 메타는 `기준일 · D-?` 형식으로 읽히게 한다.

## 마감일 규칙

- `;D`, `;d` 직접 입력은 계속 지원한다.
- 직접 입력해도 저장 시 사라지면 안 된다.

## 메모 규칙

- 통합 메모는 한 카드만 보이고, 위 탭을 바꿔가며 보는 구조를 유지한다.
- 다른 화면을 갔다 와도 보던 탭/문서를 유지한다.

## 입력창 규칙

- 여러 줄 입력 시 마지막 줄이 바로 보이도록 스크롤이 따라와야 한다.
