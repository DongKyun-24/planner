# Regression Checklist

이 문서는 웹 수정 후 확인할 것들을 가능한 한 빠짐없이 적어둔 체크리스트입니다.

## 자동 확인

- [ ] `npm run build`
- [ ] `npm run lint`

## 체크 패널

- [ ] 체크 패널이 `Task / 마감일 / Count`로 보인다.
- [ ] 각 탭의 `추가`가 현재 탭에 맞는 모달로 열린다.
- [ ] Count 추가가 브라우저 `prompt`가 아니라 전용 모달로 열린다.

## Count

- [ ] Count를 만들면 Count 탭에 보인다.
- [ ] Count가 일반 일정 본문이나 일반 D-day 목록에 섞여 보이지 않는다.
- [ ] Count에 `100`을 넣으면 `100일, 200일, 300일 ...` 기준으로 계산된다.
- [ ] Count가 10일 이내에 들어오면 Today/체크에 반영된다.
- [ ] Count와 마감일이 같은 형태로 보여서 헷갈리지 않는다.

## 마감일

- [ ] `;D`, `;d` 직접 입력이 저장 후에도 유지된다.
- [ ] 마감일 추가가 D-day가 켜진 일정 추가 흐름으로 열린다.

## 입력창

- [ ] 여러 줄 입력 시 마지막 줄이 바로 보인다.
- [ ] 직접 스크롤해야 하는 느낌이 줄어들었다.

## 메모

- [ ] 통합 메모에서 보던 탭/문서가 다른 화면 갔다 와도 유지된다.
- [ ] 첫 번째 메모로 튀지 않는다.

## 확인 우선순위

1. 체크 패널 `Task / 마감일 / Count`
2. Count 생성
3. `;D` 직접 입력
4. 통합 메모 유지
5. 여러 줄 입력창 스크롤
