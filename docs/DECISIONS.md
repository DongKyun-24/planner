# Decisions

나중에 다시 봤을 때 "왜 이렇게 했지?"를 바로 복구할 수 있게 기록한다.

## 2026-04-16 - Count를 마감일과 분리

- Count는 시작일 기준 기능이다.
- 시작일을 `1일`로 계산한다.
- 입력은 `중요 D-day 주기`에 `100, 365`처럼 넣는 방식으로 정리한다.

이유:
- 마감일 D-day와 Count는 계산 방식과 사용 기대가 다르다.
- 둘을 같은 목록에 섞어두면 개념이 계속 흐려진다.

## 2026-04-16 - 체크 패널은 3분할

- `Task / 마감일 / Count`

이유:
- Count를 마감일 옆에만 붙여두면 역할 구분이 계속 무너진다.
- 탭을 나누면 생성 흐름과 표시 규칙이 단순해진다.

## 2026-04-16 - Count 추가는 전용 모달

- 브라우저 prompt를 쓰지 않는다.
- 일정 추가창처럼 가운데 뜨는 모달에서 설정한다.

이유:
- prompt는 입력 실수가 많고 맥락 파악이 어렵다.
- Count는 설정해야 할 정보가 있어서 한 화면에서 보여주는 편이 낫다.

## 2026-04-16 - 통합 메모 상태 유지

- 통합 메모에서 보던 탭과 문서를 다른 화면 갔다 와도 유지한다.

이유:
- 자꾸 첫 번째 메모로 튀면 사용 흐름이 너무 끊긴다.

## 2026-04-16 - `;D` 직접 입력 유지

- 기존 방식은 계속 지원한다.

이유:
- 기존 입력 습관을 깨지 않기 위해서다.
