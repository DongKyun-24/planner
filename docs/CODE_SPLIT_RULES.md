# Code Split Rules

큰 파일을 한 번에 갈아엎지 않고, 기능을 만질 때마다 안전하게 분리하기 위한 기준이다.

## 기본 원칙

- 목표는 한 파일 `500줄 이하`다.
- 다만 무거운 화면 코디네이터는 `650줄 안팎`까지는 허용한다.
- 줄 수보다 중요한 건 `역할이 하나인지`다.

## 어떤 경우에 분리하나

- 한 파일이 500줄을 넘고 서로 다른 역할이 2개 이상 섞여 있을 때
- 이번 수정 범위가 120줄 이상이고, 독립 컴포넌트/유틸로 빼기 쉬울 때
- 앱/웹이 같은 규칙을 반복 구현하고 있을 때

## 웹 우선순위

1. Count / D-day 계산 유틸
2. 체크 패널 모드별 컴포넌트
3. 빠른 추가 모달 / Count 모달
4. 메모 읽기 / 편집 상태 분리

## 하지 말 것

- 한 번에 전면 리팩터링하지 않는다.
- 기능 수정과 무관한 큰 이동을 한 번에 몰아서 하지 않는다.
- 문서 없이 파일만 쪼개지 않는다.

## 작업 습관

- 새 기능을 손댈 때 "이 부분 같이 뺄 수 있나"를 먼저 본다.
- 파일을 나눴으면 `DECISIONS.md`나 `ACCEPTED_NOTES.md`에 이유를 한 줄 남긴다.
